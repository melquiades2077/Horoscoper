# 🔮 Horoscope Bot

Telegram-бот на Ruby: ежедневные гороскопы, совместимость знаков и расклады карт Таро.

Учебный проект, написанный с упором на чистую архитектуру, тесты и масштабируемость.

## 📋 Возможности

| Команда / Кнопка | Что делает |
|---|---|
| `/start` | Приветствие, показ главного меню |
| `⚙️ Настройки` | Указать дату рождения — бот определит знак |
| `🔮 Гороскоп` | Ежедневный гороскоп (детерминирован на день) |
| `💞 Совместимость` | Выбор двух знаков через кнопки, расчёт по стихиям |
| `🃏 Таро` | 3 типа расклада: карта дня / прошлое-настоящее-будущее / упрощённый кельтский крест |
| `📜 История` | Последние 5 раскладов Таро с датами |
| `/cancel` | Отменить текущий диалог |
| `/help` | Справка |

Вся навигация через кнопки (reply-клавиатура снизу экрана + inline-кнопки под сообщениями). Текстовый ввод команд работает как fallback.

## 🧪 Тесты и линтер

```
bundle exec rspec      # все тесты (модульные + интеграционные с моками)
bundle exec rubocop    # проверка стиля кода
bundle exec rake       # и то и другое сразу
```

Отчёт о покрытии создаётся в `coverage/index.html` после запуска тестов.

## 🏗 Архитектура

```
lib/horoscope_bot/
├── application.rb          # Точка сборки, запуск polling
├── command_router.rb       # Роутинг: команды, состояния, callback-и
├── keyboards.rb            # Разметка всех клавиатур в одном месте
├── zodiac.rb               # Справочник 12 знаков зодиака
├── commands/               # Паттерн Command — по файлу на команду
│   ├── base_command.rb     # Базовый класс + хелперы клавиатур
│   ├── start_command.rb
│   ├── help_command.rb
│   ├── settings_command.rb
│   ├── horoscope_command.rb
│   ├── compatibility_command.rb
│   ├── tarot_command.rb      # Inline-кнопки выбора расклада, сохранение в историю
│   ├── history_command.rb    # История последних раскладов
│   └── cancel_command.rb
├── states/                 # Машина состояний (FSM)
│   ├── user_state.rb       # Value-object состояния
│   └── state_machine.rb    # Переходы + персистентность
├── services/               # Бизнес-логика
│   ├── horoscope_generator.rb       # Детерминированный по дате
│   ├── compatibility_calculator.rb  # Матрица стихий
│   └── tarot_reader.rb              # Колода Старших Арканов + расклады
└── storage/                # Персистентный слой
    ├── json_store.rb       # Потокобезопасное key-value на JSON
    └── user_repository.rb  # Репозиторий пользователей
```

### Ключевые решения

- **Паттерн Command** — каждая команда отдельный класс, добавить новую = создать класс + одна строка в `CommandRouter`.
- **Машина состояний** — многошаговые диалоги (`/settings`, `/compatibility`, `/tarot`) управляются через `StateMachine`, состояния персистентны.
- **JSON-персистентность** — профили, состояния и история раскладов хранятся в `data/*.json`. Можно безболезненно заменить на SQLite — репозиторий абстрагирует хранилище.
- **Потокобезопасность** — `JsonStore` защищён `Mutex`.
- **Inline и reply клавиатуры** — всё навигируется кнопками, текстовый ввод работает как fallback.
- **Детерминированность** — гороскоп одинаков в течение дня (seed по дате+знаку); расклады Таро случайны, но воспроизводимы при фиксированном RNG для тестов.
